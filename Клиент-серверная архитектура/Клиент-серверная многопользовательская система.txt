Описание системы "Интернет банка":


Какие действия может выполнять пользователь:
 - запрос информации о состоянии своего счета.
 - возможность производить денежные переводы.


Для решения данной задачи выбрана многоуровневая архитектура, потому что:
 - можно выделить отдельный сервер для одновременного доступа к данным;
 - возможность добавить в систему разные типы серверов;
 - между серверами можно настроить высокоскоростное интернет соединение;
 - есть возможность организовать проверку безопасности на каждом уровне;
 - для большей надежности системы необходмо произвести резервирование серверов.


Архитектура состоит из:
 - Клиент
 - Web-сервер
 - Сервер приложений
 - Сервер базы данных
 - База данных клиентов 
 - Сервер мгновенных сообщений

Программное обеспечение:
 - Монитор обработки транзакций 


Описание работы системы:
 - Клиент. Для работы приложения пользователю понадобится веб-браузер.
   У пользователя будет тонкий клиент, который будет обеспечивать только презентационный слой.
   Интерфейс данного приложения будет сформирован средствами статического HTML.

 - Web-сервер. Необходим для формирования различных страниц сайта (например: "Главная", "Личный кабинет","Вклады" и т.д.).

 - Сервер приложений. Отвечает за бизнес-логику. 
   Принимает и обрабатывает запросы от пользователей.
   Определяет запрос на нужный сервер и возвращает ответ пользователю. 

 - Сервер базы данных. Взаимодействует с базой данных клиентов.
   Принимает различные запросы на языке SQL.

 - База данных клиентов. Содержит в себе информацию о клиентах (ФИО, дата рождения, счета и т.д.).

 - Сервер мгновенных сообщений. Отвечает за доставку сообщений между пользователем и оператором банка или за организацию сообщений чат-бота.

 - Монитор обработки транзакций. Отвечает за транзации, совершаемые при переводе средств со счета клиента на другой счет.
   Для обеспечения выполнения транзакций осуществляется взаимодействие операций на нескольких серверах, чтобы операция была выполнена успешно либо не выполнена вовсе.
   Возможно взаимодействие осуществляется и со сторонними системами при переводе средств с одного счета в одном банка на второй счет другого банка.


Решение проблем производительности, безопасности и надежности:
 - если организовать между серверами высокоскоростное интернет соединение, то производительность увеличится;
 - проблему надежности можно решить резервными серверами и базами данных, на случай отказа работоспособности первых или, например, выключения электроэнергии;
 - проблема безопасности решена тем, что пользователь не имеет прямого доступа к серверу базы данных.
